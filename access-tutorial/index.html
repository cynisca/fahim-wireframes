Module 1

The Big Picture

Microsoft Access is a desktop relational database that comes with Microsoft 365. Think of it as Excel's more structured cousin—instead of one giant spreadsheet, your data lives in separate, linked tables with strict rules.

Why do MBAs learn Access?

It teaches you to think in terms of structured data: how businesses store customer records, orders, inventory, and financials so they can query and report on them. The same concepts apply to SQL Server, Oracle, Salesforce—anywhere data lives in a real company.

The four objects in Access

Tables hold the raw data. Queries filter, sort, and calculate. Forms give users a friendly way to enter data. Reports format data for printing or PDFs.

💡 Mental model
Tables = spreadsheet tabs  |  Queries = Excel filters on steroids  |  Forms = data-entry screens  |  Reports = formatted printouts

Tables & Data Types

A table is a grid of rows (records) and columns (fields). Every table needs a Primary Key—a field that uniquely identifies each row (like a student ID or order number).

Common data types

Data TypeUse ForExample
Short TextNames, codes, labels (up to 255 chars)John Smith
Long TextNotes, descriptionsCustomer feedback paragraph
NumberQuantities, scores42
CurrencyMoney values$1,299.99
Date/TimeDates, timestamps03/15/2026
Yes/NoBoolean flags☑ Active
AutoNumberAuto-incrementing primary keys1, 2, 3…

Field properties to know

Field Size — limits length of text/number. Required — prevents blanks. Default Value — auto-fills a starting value. Validation Rule — enforces constraints (e.g., >0 means only positive numbers). Input Mask — formats input (e.g., phone number pattern).

Module 3

Relationships

The whole point of a relational database is that tables are linked. Instead of repeating a customer's name on every order row, you store the customer once and reference their ID.

The three relationship types

TypeMeaningExample
One-to-Many (1:∞)One record in Table A matches many in Table BOne customer → many orders
One-to-One (1:1)One record matches exactly one in another tableEmployee → employee details
Many-to-Many (∞:∞)Requires a junction table in betweenStudents ↔ Classes (via Enrollments)

How to create a relationship

Go to Database Tools → Relationships. Drag the primary key from one table onto the matching foreign key in another. Check Enforce Referential Integrity — this prevents orphan records (e.g., an order pointing to a customer that doesn't exist).

Module 4

Queries

Queries are the most powerful part of Access. They let you filter, sort, calculate, and combine data from multiple tables.

Select Query (the most common)

Go to Create → Query Design. Add tables, drag fields into the grid, set criteria and sort order. This creates a Select Query—it reads data without changing it.

Criteria syntax cheat sheet

What you wantCriteria
Exact match"New York"
Not equalNot "New York"
Greater than $100>100
Between two datesBetween #1/1/2026# And #3/31/2026#
Starts with "S"Like "S*"
Is blank / not blankIs Null / Is Not Null
Multiple values"NY" Or "CA" Or "TX"

Calculated fields

You can create new columns in the query grid. Just type an expression in a blank column:

Tax: [TotalAmount] * 0.08
GrandTotal: [TotalAmount] + [TotalAmount] * 0.08
FullName: [FirstName] & " " & [LastName]

Totals queries (aggregation)

Click the Σ Totals button in query design. This adds a "Total" row where you pick: Group By, Sum, Avg, Count, Min, Max.

Example: "Total spend per customer" → Group By CustomerID, Sum of TotalAmount.

Other query types

Action queries modify data: Update (change values), Append (insert rows), Delete (remove rows), Make-Table (create a new table from results). Use with caution—they change your data permanently.

Parameter queries prompt the user for input at runtime. Put brackets around a prompt in the criteria: [Enter city name:]

🏋️ Exercise 3 — Build three queries

Query A: qryHighValueOrders

This query will show all orders over $50, along with the customer's name.

Step 1. Click the Create tab in the ribbon, then click Query Design.

Step 2. The "Show Table" dialog appears. Double-click tblCustomers, then double-click tblOrders. Close the dialog. You'll see both tables on the canvas with a line connecting them (Access knows they're related).

Step 3. Now you need to pick which fields to show. In the tblCustomers box, double-click FirstName — it drops into the grid at the bottom. Double-click LastName. Then in the tblOrders box, double-click OrderDate and TotalAmount.

Step 4. In the grid at the bottom, find the TotalAmount column. In the Criteria row (scroll down in the grid if you don't see it), type: >50

Step 5. Click the Run button in the ribbon (it looks like a red exclamation mark !). You should see only orders over $50 with customer names.

Step 6. Save it: press Ctrl + S. Name it qryHighValueOrders. Click OK.

Query B: qrySpendByCustomer

This query will show the total amount each customer has spent across all their orders.

Step 1. Click Create → Query Design. Add both tables again (same as before). Close the Show Table dialog.

Step 2. Double-click these fields to add them to the grid: CustomerID (from tblCustomers), FirstName, LastName, and TotalAmount (from tblOrders).

Step 3. Now we need to turn on Totals. In the ribbon, look for the Σ Totals button (it's in the "Show/Hide" group on the Design tab) and click it. A new row called "Total" appears in the grid.

Step 4. By default, every column says "Group By" in the Total row. That's correct for CustomerID, FirstName, and LastName. But for TotalAmount, click the dropdown in the Total row and change it from "Group By" to Sum.

Step 5. Click Run (!). You should see one row per customer with their total spend.

Step 6. Save as qrySpendByCustomer.

Query C: qryOrdersByCity

This query will ask you to type in a city name when you run it, then show only orders from customers in that city.

Step 1. Click Create → Query Design. Add both tables. Close the dialog.

Step 2. Add these fields to the grid: FirstName, LastName, City (from tblCustomers), OrderDate, TotalAmount (from tblOrders).

Step 3. In the Criteria row under City, type exactly this (including the square brackets): [Enter a city:]

Step 4. Click Run (!). A small dialog box pops up asking "Enter a city:" — type San Jose and click OK. You should see only orders from San Jose customers.

💡 What just happened?
The square brackets [ ] tell Access: "don't look for this literally — instead, ask the user to type something in." Whatever they type gets used as the filter. This is called a parameter query.

Module 5

Forms

Forms provide a user-friendly interface for data entry instead of editing tables directly. In a real business, end users would never touch the raw tables—they'd use forms.

Quick way to create a form

Select a table in the navigation pane, then click Create → Form. Access auto-generates a layout. Switch to Layout View to rearrange fields, resize, add logos, and change colors.

Key form concepts

Bound vs. Unbound controls: A bound text box is linked to a table field (edits save automatically). An unbound control is standalone (like a label or calculated field).

Subforms: Embed a related table inside a form. Example: a customer form with an orders subform showing all their orders at the bottom.

Combo box: A dropdown that lets users pick from a list (e.g., select a customer from a dropdown instead of typing the ID manually). Use the wizard to set it up.

Module 6

Reports

Reports format data for printing or PDF export. They're read-only—users can't edit data through a report.

Creating a report

Select a query (or table) → Create → Report. Access generates a report. Use Layout View to adjust column widths and add headers/footers.

Grouping & sorting

In Design or Layout View, click Group & Sort. You can group by a field (e.g., group orders by customer) and add subtotals at each group break. The report footer can show grand totals.

Calculated controls in reports

=Sum([TotalAmount])        ← grand total in report footer
=Count(*)                  ← number of records
=Now()                     ← current date/time
="Page " & [Page] & " of " & [Pages]  ← page numbers

Module 7

SQL Basics

Every query in Access has a SQL View behind it. Understanding basic SQL makes you dangerous in any data role.

⚡ Pro tip
In any query, click View → SQL View to see (and edit) the SQL. This is the fastest way to learn SQL—build queries in design view, then read the generated SQL.

Core SQL statements

-- Select specific columns with a filter
SELECT FirstName, LastName, City
FROM tblCustomers
WHERE City = "New York"
ORDER BY LastName;

-- Join two tables
SELECT c.FirstName, c.LastName, o.OrderDate, o.TotalAmount
FROM tblCustomers AS c
INNER JOIN tblOrders AS o
ON c.CustomerID = o.CustomerID
WHERE o.TotalAmount > 50;

-- Aggregation with GROUP BY
SELECT c.LastName, SUM(o.TotalAmount) AS TotalSpend
FROM tblCustomers AS c
INNER JOIN tblOrders AS o
ON c.CustomerID = o.CustomerID
GROUP BY c.LastName
HAVING SUM(o.TotalAmount) > 100;

SQL keyword cheat sheet

KeywordPurpose
SELECTWhich columns to show
FROMWhich table(s)
WHEREFilter rows
ORDER BYSort results
GROUP BYAggregate into groups
HAVINGFilter groups (like WHERE for aggregates)
INNER JOIN … ONCombine matching rows from two tables
LEFT JOIN … ONAll rows from left table + matches from right
🏋️ Exercise 6 — Write SQL by hand

How to open SQL View

Step 1. Click the Create tab, then click Query Design.

Step 2. The "Show Table" dialog appears — just close it without adding any tables (click Close). You'll see an empty query grid.

Step 3. In the bottom-left corner of the Access window, you'll see small view icons, OR you can go to View in the ribbon → click the dropdown arrow → choose SQL View. You now see a blank text area where you can type SQL directly.

Query A: Customers sorted by city, then last name

Step 4. Delete anything in the SQL text area, then type:

SELECT FirstName, LastName, City, Email
FROM tblCustomers
ORDER BY City, LastName;

Step 5. Click Run (!) in the ribbon. You should see all your customers sorted alphabetically by city, and within each city, sorted by last name.

Step 6. Save as qryCustomersByCity.

Query B: Recent orders (last 30 days)

Step 7. Create another new query: Create → Query Design. Close the Show Table dialog. Switch to SQL View.

Step 8. Type:

SELECT c.FirstName, c.LastName, o.OrderDate, o.TotalAmount
FROM tblCustomers AS c
INNER JOIN tblOrders AS o
ON c.CustomerID = o.CustomerID
WHERE o.OrderDate >= Date() - 30
ORDER BY o.OrderDate DESC;

Step 9. Click Run (!). This shows orders from the past 30 days, newest first. (If your sample dates are older than 30 days, you might see no results — that's OK, it means the SQL is working correctly. Try changing 30 to 365 to see results.)

Step 10. Save as qryRecentOrders.

Query C: Count orders per customer

Step 11. Create one more: Create → Query Design. Close Show Table. Switch to SQL View.

Step 12. Type:

SELECT c.FirstName, c.LastName, COUNT(o.OrderID) AS NumberOfOrders
FROM tblCustomers AS c
INNER JOIN tblOrders AS o
ON c.CustomerID = o.CustomerID
GROUP BY c.FirstName, c.LastName
ORDER BY COUNT(o.OrderID) DESC;

Step 13. Click Run (!). You should see each customer with how many orders they've placed, with the most frequent buyer at the top.

Step 14. Save as qryOrderCounts.
